RavenPoint API Guide

Welcome to RavenPoint!

This guide aims to help you get up and running with RavenPoint ASAFP.

Main Features

RavenPoint has two main features:

  1. RavenPoint Admin Panel
  2. RavenPoint API

RavenPoint API

The API emulates the SharePoint's REST API, specifically the endpoints that enable interaction with Lists. The (1) syntax for URL parameters (Open Data Protocol a.k.a OData), (2) required headers, and (3) responses are designed to mimic those used on the SharePoint REST API as closely as possible. This provides Team Raven Stack developers with a seamless experience in developing React apps in a production environment.

RavenPoint Admin Panel

The Admin Panel is what you're looking at. It allows you to add and delete (1) tables and (2) relationships between the tables.

Loading Data

First, it's important to note that RavenPoint only accepts data in the form of CSV files. We don't expect any non-tabular data since this is meant to emulate SharePoint's REST API for access to Lists.

Second, ensure the first column in your dataset is a column named Id containing unique integers. This is because (1) it is required to accurately emulate SharePoint, which has a default column of the same name, and (2) RavenPoint automatically processes the leftmost column as the primary key.

Third, load your data on the Tables page:

  1. Hit the Add Table button to launch a pop-up window with a form.
  2. Input a table name. Think carefully! This name will be used to generate a database table name. Characters other than alphanumerics and spaces are strongly not encouraged. Please also follow typical naming conventions (e.g. don't put a number in front). As an example of how the name will be transformed, DC Datasets will be stored in the backend database as dc_datasets.
  3. Hit Submit and wait. Do not click the Submit button in quick succession. Doing so may result in an error writing to the database.
  4. If the data is loaded successfully, you will be re-directed back to the Admin Panel home page (Tables view) and a success message will appear. If that does not happen, check your data and try again.
